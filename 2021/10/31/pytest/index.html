  • Matching

    In this case, use pytest test.py -k method will deselect the last test, but only select the first 2

  • Mark

    In this case, use pytest test.py -m one/two will specify the test with marker one or two.

use @pytest.mark.skip will skip the test

use @pytest.mark.xfail even the test fails, it will not be counted

  • Class In this case, you can directly ues pytest test.py
  1. Fixture The fixture function will run every time before running any tests

There is also a scope parameter that could determine the moment when fixture will be run, e.g. the start of each session.

  1. Paramentrized Tests This will call 2 tests for two sets of parameters in one session.
